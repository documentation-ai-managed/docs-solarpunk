## Upgrading Solarpunk

Follow these steps to upgrade to the latest version safely.

<Steps>
  <Step title="Backup Data" icon="database">
    Export your configurations and agent states from the dashboard.

    <CodeGroup tabs="CLI,Dashboard">
    ````bash
    solarpunk export --backup ./my-backup
    ````
    ````bash
    # Use dashboard UI: Settings > Export
    ````
    </CodeGroup>
  </Step>

  <Step title="Download Update" icon="download">
    Fetch the latest release from your account.

    ````bash
    solarpunk update --version latest
    ````
  </Step>

  <Step title="Verify Migration" icon="check-circle">
    Run migration checks and review changelogs.

    <Callout kind="tip">
      For v1.2.0, update your `{config.yaml}` file to the new schema.
    </Callout>
  </Step>

  <Step title="Restart Services" icon="refresh-cw">
    Restart Ubik agents to apply changes.

    ````bash
    solarpunk restart --all
    ````
  </Step>
</Steps>

<Expandable title="Migration Notes for Breaking Changes" default-open="false">

If upgrading from v0.x, migrate your config files manually:

```yaml
# Old format (JSON-like)
agents:
  - name: "ubik"

# New YAML format (v1.0.0+)
agents:
  - name: ubik
    model: local-llm
```

Review full migration guide in the [dashboard](https://dashboard.example.com/migrations).

</Expandable>
